# No introducir tecnologia nueva sin aprobacion

**Regla:** No proponer migraciones de stack, nuevas librerias, ni cambios estructurales que el owner no haya pedido. Si surge una mejora tecnica, mencionarla brevemente como sugerencia y seguir.

**Why:** El owner aclaro que no entiende la parte tecnica y que esta listo para lanzar. Confundio "NO introducir Vercel AI SDK" (advertencia) con "VOY a introducir Vercel AI SDK" (propuesta) — eso indica que mencionar tech que no aplica genera ruido. Tambien confirmo que Lemon se queda y no quiere cambiar.

**How to apply:**
- Para usuarios no-tecnicos, dar respuestas en lenguaje simple.
- No listar "alternativas tecnicas" salvo que las pida.
- Cuando escriba advertencias en CLAUDE.md (seccion "No Hacer"), aclarar que son recordatorios para mi (Claude), no propuestas para el owner.
- Priorizar progreso al lanzamiento sobre limpieza tecnica.
